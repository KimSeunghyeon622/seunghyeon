# 테스트 계획서 작성 스킬

포괄적인 테스트 계획서를 작성합니다. AI/시스템이 자체적으로 테스트를 수행할 수 있도록 상세 계획을 수립합니다.

## 실행 지침

1. PRD(`docs/PRD.md`), TRD(`docs/TRD.md`), API 명세 참조
2. 테스트 전략 및 시나리오 정의
3. `docs/Test-Plan.md`에 저장

## 테스트 피라미드

```
      /\
     /  \     E2E (10%) - 핵심 플로우만
    /----\
   /      \   Integration (20%) - API, DB 연동
  /--------\
 /          \ Unit (70%) - 함수, 컴포넌트
--------------
```

## 테스트 유형별 도구

| 유형 | 도구 | 커버리지 목표 |
|------|------|-------------|
| Unit | Jest | 80% |
| Integration | Jest + Supertest | 60% |
| Component | React Testing Library | 70% |
| E2E | Playwright | 핵심 플로우 |

## 테스트 케이스 형식

```markdown
### 테스트 대상: [함수/컴포넌트/API명]

| 케이스 | 입력 | 예상 결과 | 우선순위 |
|--------|------|----------|----------|
| 정상 케이스 | 유효 데이터 | 성공 | High |
| 엣지 케이스 | 경계값 | 적절한 처리 | Medium |
| 에러 케이스 | 잘못된 데이터 | 에러 메시지 | High |
```

## 핵심 테스트 시나리오

### 필수 테스트 (Must-have)
- [ ] 회원가입 → 로그인 플로우
- [ ] 예약 생성 → 확정 → 완료
- [ ] 예약 취소 및 환불

### 보안 테스트
- [ ] 인증 없이 보호된 API 접근
- [ ] 다른 사용자 데이터 접근 시도
- [ ] SQL 인젝션, XSS 방어
