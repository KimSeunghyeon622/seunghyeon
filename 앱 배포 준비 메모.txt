중요 개념
- Android 패키지 ID / iOS 번들 ID는 앱의 고유 식별자이며 스토어 등록 및 업데이트에 중요함
- OTA(EAS Update)는 JS/UI 변경에만 사용 가능, 권한/네이티브 SDK 변경은 스토어 업데이트 필요

현재까지 합의/결론
- 계정 준비 전에는 패키지/번들 ID 확정 보류
- 배포 준비는 문서 정리 후 다시 진행 예정

추가로 필요했던 확인 질문(미답)
1) Android 패키지 ID를 com.saveit.app 그대로 사용할지
2) iOS 번들 ID 값(예: com.bbcompany.saveit)
3) 앱 표시 이름(스토어/아이콘용)
4) 푸시 수신 대상/트리거 상세(누가 받는지, 어떤 이벤트에서 발송되는지, 빈도)

앞으로 진행 순서(요약)
1) Apple/Google 개발자 계정 준비
2) 패키지 ID/번들 ID/앱 이름 확정
3) EAS 설정 및 빌드 준비
4) 푸시 키(Firebase FCM, APNs) 준비 + EAS 등록
5) 스토어 빌드/심사/공개
6) 운영: OTA + 스토어 업데이트 구분하여 배포

나(사용자) 혼자 준비할 수 있는 것들
  1. 스토어 계정 준비
  2. Apple Developer Program 가입
      - 개인도 가능, 연 $99
      - 신분증/결제수단 준비 필요
  3. Google Play Console 가입
      - 1회 $25
      - 결제수단 + 개발자 프로필 정보 입력
  4. 앱 식별자/브랜딩 확정
  5. 앱 표시 이름(스토어에 노출되는 이름)
  6-7. iOS 번들 ID/ Android 패키지 ID는 com.saveit.app 

  8. 앱 아이콘/스플래시 이미지 기본 리소스 준비
  9. 스토어 등록용 소재 준비
  10. 앱 설명(짧은/긴 설명) 초안
  11. 스토어 스크린샷(최소 4~6장 권장)
  12. 개인정보처리방침 URL (이미 있음)
  13. 고객 문의용 메일 주소/고객센터 정보
  14. 푸시 정책/운영 정의
  15. 누가 푸시를 받는지 (고객/사장님/둘 다)
  16. 어떤 이벤트에서 보내는지 (거래 발생/마감 할인 등)
  17. 빈도 제한 정책 (1일 최대 몇 건 등)
  18. 배포 후 운영 준비
  19. 긴급 수정 대응 방식 정의 (OTA로 할 것 vs 스토어 업데이트)
  20. 내부 테스트 그룹(테스트 기기/테스터 명단)
  21. 앱 심사 시 자주 보는 정책 항목 정리(환불/개인정보)
