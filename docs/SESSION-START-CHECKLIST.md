# 세션 시작 체크리스트 (Session Start Checklist)

> **목적**: 새로운 Claude 세션 시작 시 반드시 확인해야 할 항목들을 체크리스트로 제공  
> **사용 시점**: Claude와 대화를 시작할 때마다

---

## 🚨 필수 확인 항목 (MANDATORY - 자동 실행)

**⚠️ CRITICAL: 사용자가 요청하지 않아도 자동으로 실행되어야 합니다.**

새로운 세션을 시작할 때, Claude/Cursor는 다음 항목들을 **사용자 요청 없이 자동으로 확인**해야 합니다:

### 1. 툴 교체 이력 확인
- [ ] `docs/TOOL-SWITCH-LOG.md` 파일 읽기
- [ ] 이전 툴(Cursor/Claude)에서 완료된 작업 확인
- [ ] 진행 중인 작업 상태 파악
- [ ] 다음 작업 계획 확인
- [ ] 주의사항 및 특이사항 확인

### 2. 프로젝트 상태 확인
- [ ] `docs/WORK-LOG.md`의 "최근 업데이트" 섹션 읽기
- [ ] 현재 프로젝트 진행 상태 파악
- [ ] 최근 완료된 작업 확인
- [ ] 알려진 이슈 확인

### 3. 프로젝트 구조 이해
- [ ] `docs/CURSOR-ONBOARDING.md` 읽기
- [ ] 프로젝트 구조 파악
- [ ] 주요 문서 위치 확인
- [ ] 개발 환경 설정 확인

### 4. 팀 운영 규칙 확인
- [ ] `CLAUDE.md` 읽기
- [ ] 팀장 역할 및 원칙 확인
- [ ] 팀원(에이전트/스킬) 구조 확인
- [ ] 작업 위임 원칙 확인

---

## 📋 확인 후 보고 형식

Claude는 다음 세션 시작 시 다음 형식으로 확인 완료를 보고해야 합니다:

```markdown
## ✅ 세션 시작 확인 완료

### 읽은 문서
- ✅ `docs/TOOL-SWITCH-LOG.md` - [요약]
- ✅ `docs/WORK-LOG.md` - [최근 업데이트 요약]
- ✅ `docs/CURSOR-ONBOARDING.md` - [프로젝트 구조 확인]
- ✅ `CLAUDE.md` - [팀 운영 규칙 확인]

### 현재 프로젝트 상태
- **진행률**: [%]
- **최근 완료 작업**: [작업 목록]
- **진행 중인 작업**: [작업 목록 또는 없음]

### 다음 작업 계획
- [작업 1]
- [작업 2]

### 주의사항
- [주의사항 1 또는 없음]
- [주의사항 2 또는 없음]

### 이어서 진행할 작업
- [작업 내용 또는 없음]
```

## 🔍 빠른 확인 방법

### 파일 존재 여부 확인
```bash
# TOOL-SWITCH-LOG.md 확인
ls docs/TOOL-SWITCH-LOG.md

# WORK-LOG.md 확인
ls docs/WORK-LOG.md

# 최근 수정 시간 확인
stat docs/TOOL-SWITCH-LOG.md
stat docs/WORK-LOG.md
```

### 최근 업데이트 확인
- `TOOL-SWITCH-LOG.md`의 "마지막 업데이트" 날짜 확인
- `WORK-LOG.md`의 "마지막 업데이트" 날짜 확인
- 최근 업데이트가 24시간 이내라면 반드시 읽기

---

## ⚠️ 중요 사항

1. **절대 건너뛰지 마세요**: 이 체크리스트는 선택사항이 아닙니다.
2. **사용자가 요청하지 않아도**: 새로운 세션 시작 시 자동으로 확인해야 합니다.
3. **불확실하면 읽으세요**: 파일이 언제 업데이트되었는지 불확실하면 읽는 것이 안전합니다.
4. **요약 보고 필수**: 읽은 후 반드시 요약 보고를 해야 합니다.

---

## 💡 자동 실행 시나리오

### 시나리오 1: 새로운 세션 시작 (사용자 요청 없음)
```
사용자: "안녕" 또는 "시작해줘"

Claude/Cursor 자동 응답:
1. "프로젝트 상태를 자동으로 확인하겠습니다."
2. TOOL-SWITCH-LOG.md, WORK-LOG.md 자동 읽기
3. 확인 완료 보고 자동 수행
4. 다음 작업 제안
```

### 시나리오 2: 특정 작업 요청
```
사용자: "장바구니 기능 추가해줘"

Claude/Cursor 자동 응답:
1. "네, 먼저 현재 프로젝트 상태를 자동으로 확인하겠습니다."
2. TOOL-SWITCH-LOG.md, WORK-LOG.md 빠르게 자동 확인
3. 관련 작업 이력 확인
4. 작업 시작
```

### 시나리오 3: 사용자가 명시적으로 확인 요청
```
사용자: "이력 확인이 되어있나요?"

Claude/Cursor 응답:
1. "네, 자동으로 확인했습니다" 또는 "지금 확인하겠습니다."
2. 확인 내용 요약
3. 다음 단계 제시
```

**중요**: 시나리오 1과 2는 사용자가 요청하지 않아도 자동으로 실행되어야 합니다.
