# UI 디자인: 판매상품 관리 화면 개선

## 1. 화면 목적 및 사용자 시나리오

### 목적
- 기존: 판매상품 관리 진입 시 바로 상품 등록 폼이 표시됨
- 개선: 상품 리스트를 먼저 표시하고, "신규 상품등록" 버튼을 통해 등록 폼으로 이동

### 사용자 시나리오
1. 업주가 "판매상품 관리" 메뉴 진입
2. **현재 등록된 상품 리스트** 확인
3. 상단의 "신규 상품등록" 버튼 클릭
4. 상품 등록 폼으로 이동
5. 상품 등록 완료 후 리스트로 돌아옴

## 3. 컴포넌트 명세

### 3.1 헤더
- 뒤로가기 버튼 (←)
- 타이틀: "상품 관리"
- 알림 아이콘 (선택사항)

### 3.2 신규 상품등록 버튼
- **위치**: 리스트 최상단
- **스타일**:
  - 배경: #00D563 (primary green)
  - 텍스트: 흰색, bold
  - 아이콘: ➕ 또는 🛒
  - 패딩: 16px vertical
  - 모서리: 12px radius
- **인터랙션**: 클릭 시 상품 등록 폼으로 이동

### 3.3 상품 카드
- **레이아웃**:
  - 좌측: 상품 이미지 (80x80)
  - 우측: 상품 정보
- **표시 정보**:
  - 상품명
  - 할인가 (원가 대비 할인율)
  - 재고 수량
  - 판매 상태 (판매중/판매중지)
- **액션 버튼**:
  - 수정: 상품 수정 모드로 전환
  - 삭제: 확인 후 삭제
  - 판매중지/재판매: 토글

### 3.4 빈 상태
- **표시 조건**: 등록된 상품이 없을 때
- **내용**:
  - 아이콘: 📦
  - 메시지: "등록된 상품이 없습니다"
  - 서브메시지: "상품을 등록해주세요"
  - CTA: "첫 상품 등록하기" 버튼

## 6. 인터랙션 정의

### 6.1 신규 상품등록 버튼
- **탭**: 상품 등록 폼 화면으로 전환 (mode: 'create')
- **피드백**: 버튼 누름 효과 (opacity 0.8)

### 6.2 상품 카드
- **탭**: 상품 상세/수정 모드로 전환
- **스와이프 (옵션)**: 삭제 버튼 노출

### 6.3 수정 버튼
- **탭**: 해당 상품 정보가 채워진 등록 폼으로 전환 (mode: 'edit')

### 6.4 삭제 버튼
- **탭**: Alert 확인 다이얼로그
  - 제목: "상품 삭제"
  - 내용: "[상품명]을 삭제하시겠습니까?"
  - 버튼: "취소" / "삭제"

### 6.5 판매중지/재판매 버튼
- **탭**: 즉시 상태 토글
- **피드백**: 상태 변경 완료 Toast

## 7. 구현 참고사항

### 7.1 화면 모드 관리
```typescript
type ViewMode = 'list' | 'create' | 'edit';

const [viewMode, setViewMode] = useState<ViewMode>('list');
const [editingProduct, setEditingProduct] = useState<Product | null>(null);
```

### 7.2 상품 데이터 구조
```typescript
interface Product {
  id: string;
  name: string;
  category: string;
  original_price: number;
  discounted_price: number;
  stock_quantity: number;
  image_url: string | null;
  is_active: boolean;
  manufactured_date: string | null;
  expiry_date: string | null;
}
```

### 7.3 화면 전환 로직
- `viewMode === 'list'`: 상품 리스트 표시
- `viewMode === 'create'`: 새 상품 등록 폼 표시
- `viewMode === 'edit'`: 선택된 상품 수정 폼 표시

### 7.4 기존 코드 활용
- 상품 등록 폼은 기존 `StoreProductManagement.tsx` 코드 재활용
- 리스트 화면만 새로 추가
