## 1. 제품 비전

### 1.1 해결하고자 하는 핵심 문제

| 문제 | 현재 상황 | 목표 상태 |
|------|----------|----------|
| **판매 인사이트 부재** | 오늘/주/월 매출 숫자만 확인 가능 | 시간대별, 요일별, 고객별 심층 분석 |
| **수익 구조 불투명** | 총매출만 표시, 수수료/순수익 구분 없음 | 수수료 차감 후 실수익 명확히 파악 |
| **재고 효율성 모름** | 등록 재고 대비 판매율 확인 불가 | 미판매 원인 분석, 최적 재고량 파악 |
| **고객 이해 부족** | 누가 단골인지, 신규인지 모름 | 고객 세그먼트별 구매 패턴 파악 |
| **성장 추적 불가** | 전주/전월 대비 비교 없음 | 성장률 추이 한눈에 확인 |

### 1.2 제품 솔루션

**"사장님 전용 판매 분석 대시보드"**

소규모 업체 사장님이 복잡한 분석 도구 없이도 매출 패턴을 이해하고, 더 나은 판매 결정을 내릴 수 있도록 돕는 직관적인 판매 관리 기능.

### 1.3 성공의 정의 [가정]

| 지표 | 목표치 | 측정 방법 |
|------|--------|----------|
| 기능 진입률 | 일 1회 이상 (활성 업주 기준) | 화면 진입 로그 |
| 평균 체류 시간 | 2분 이상 | 세션 분석 |
| 업주 만족도 | 4.0/5.0 이상 | 인앱 피드백 |
| 재고 폐기율 감소 | 10% 감소 | 미판매 재고 추적 |

## 2. 목표 및 KPI

### 2.1 비즈니스 목표

| 목표 | 설명 | 연결 지표 |

| **고객 재구매율 향상** | 단골 관리 기능으로 충성 고객 확보 | 고객 재구매율 |

## 4. 핵심 기능

### 4.1 기능 개요

```
[판매 관리 메인 화면]
    |
    +-- [1. 매출 요약] -------- 오늘/주/월 매출, 성장률
    |
    +-- [2. 시간대별 분석] ---- 언제 잘 팔리나
    |
    +-- [3. 요일별 분석] ------ 어떤 요일이 바쁜가
    |
    +-- [4. 상품 분석] -------- 인기/비인기 상품
    |
    +-- [5. 고객 분석] -------- 신규 vs 재구매, 단골
    |
    +-- [6. 재고 효율성] ------ 등록 대비 판매율
    |
    +-- [7. 수익 분석] -------- 수수료/순수익
    |
    +-- [8. 취소/노쇼 분석] --- 취소율, 노쇼율
```

### 4.2 기능별 상세

#### 4.2.1 매출 요약 (기존 강화)

| 항목 | 현재 | 개선 |
|------|------|------|
| 오늘 매출 | 금액만 표시 | + 전일 대비 증감률 |
| 주간 매출 | 금액만 표시 | + 전주 대비 증감률 |
| 월간 매출 | 금액만 표시 | + 전월 대비 증감률 |
| 일별 추이 | 리스트 형태 | 시각적 막대 그래프 |

#### 4.2.2 시간대별 판매 분석 (신규)

| 기능 | 설명 | 데이터 소스 |
|------|------|------------|
| 피크 타임 표시 | 가장 판매 많은 시간대 강조 | `reservations.created_at` |
| 시간대별 막대 그래프 | 09-12, 12-15, 15-18, 18-21 | 4시간 단위 집계 |
| 추천 메시지 | "오후 6-9시가 가장 바빠요!" | 자동 생성 |

#### 4.2.3 요일별 판매 분석 (신규)

| 기능 | 설명 | 데이터 소스 |
|------|------|------------|
| 요일별 매출 | 월~일 막대 그래프 | `reservations.created_at` |
| 바쁜 날 표시 | 상위 2개 요일 강조 | 평균 대비 계산 |
| 휴무일 추천 | "화요일이 가장 한가해요" | 자동 분석 |

#### 4.2.4 상품 분석 (기존 강화)

| 항목 | 현재 | 개선 |
|------|------|------|
| 인기 상품 TOP 5 | 있음 | 유지 |
| 비인기 상품 | 없음 | 하위 3개 표시 + 개선 제안 |
| 상품별 수익률 | 없음 | 상품별 순수익 표시 |
| 카테고리별 분석 | 없음 | 빵/도시락/음료 등 |

#### 4.2.5 고객 분석 (신규)

| 기능 | 설명 | 데이터 소스 |
|------|------|------------|
| 신규 vs 재구매 | 이번 달 비율 표시 | `reservations` + `consumers` |
| 단골 비율 | 3회 이상 구매 고객 비율 | 구매 횟수 집계 |
| 단골 TOP 5 | 가장 많이 방문한 고객 | 닉네임 + 구매 횟수 |
| 평균 객단가 | 고객 1인당 평균 결제 금액 | `total_amount` 평균 |

#### 4.2.6 재고 효율성 분석 (신규)

| 기능 | 설명 | 데이터 소스 |
|------|------|------------|
| 등록 대비 판매율 | 등록 재고 중 판매된 비율 | `products.stock_quantity` + `reservations` |
| 미판매 분석 | 팔리지 않은 상품/시간대 | 예약 없는 상품 추적 |
| 적정 재고 추천 | "평균 판매량 기준 5개 등록 권장" | 자동 계산 [가정] |

#### 4.2.7 수익 분석 (신규, 핵심)

| 항목 | 설명 | 계산식 |
|------|------|--------|
| 총매출 | 예약 확정 금액 합계 | SUM(total_amount) |
| 수수료 합계 | 플랫폼에 지불한 수수료 | SUM(commission_amount) |
| 순수익 | 실제 수령 금액 | 총매출 - 수수료 |
| 순수익률 | 순수익 / 총매출 * 100 | 85% (수수료 15%) |

#### 4.2.8 취소/노쇼 분석 (신규)

| 기능 | 설명 | 데이터 소스 |
|------|------|------------|
| 취소율 | 전체 예약 대비 취소 비율 | `status = 'cancelled'` |
| 노쇼율 | 확정 예약 대비 미픽업 비율 | `status = 'no_show'` |
| 취소 사유 분석 | 고객 취소 vs 업주 취소 | `cancel_reason` |
| 추이 그래프 | 주간 취소/노쇼 추이 | 일별 집계 |

### 4.3 우선순위 분류

| 우선순위 | 기능 | 이유 |
|----------|------|------|
| **Must-have (P0)** | 매출 요약 강화 (성장률) | 기존 기능 개선, 즉시 가치 |
| **Must-have (P0)** | 수익 분석 | 사장님 핵심 관심사 |
| **Must-have (P0)** | 시간대별 분석 | 운영 결정에 직결 |
| **Should-have (P1)** | 요일별 분석 | 휴무일/영업 계획 |
| **Should-have (P1)** | 상품 분석 강화 | 메뉴 결정 지원 |
| **Should-have (P1)** | 고객 분석 | 단골 관리 니즈 |
| **Nice-to-have (P2)** | 재고 효율성 | 데이터 축적 필요 |
| **Nice-to-have (P2)** | 취소/노쇼 분석 | 발생 빈도 낮음 [가정] |

| 단골 TOP 5 | 단골 관리 | 업주 | 고객 분석 탭 | consumers.nickname, 구매횟수 | 3회 이상 = 단골 | 닉네임 프라이버시 | M | P1 |
| 재고 판매율 | 효율성 개선 | 업주 | 재고 분석 탭 | products.stock_quantity, reservations.quantity | 일별 등록 재고 추적 필요 | 재고 이력 테이블 필요 | H | P2 |
| 취소율/노쇼율 | 운영 개선 | 업주 | 취소 분석 탭 | reservations.status, cancel_reason | status별 집계 | 발생 빈도 낮을 수 있음 | M | P2 |
