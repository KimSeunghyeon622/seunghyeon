# Expo 터널 모드 사용 가이드

## 터널 모드란?

터널 모드는 Expo의 서버를 통해 개발 서버에 연결하는 방식입니다. 일반 모드(LAN)와 달리 네트워크 제약 없이 어디서든 연결할 수 있습니다.

---

## 터널 모드 vs 일반 모드 비교

| 항목 | 터널 모드 | 일반 모드 (LAN) |
|------|----------|----------------|
| **네트워크 요구사항** | 없음 (다른 네트워크 가능) | 같은 Wi-Fi 필요 |
| **방화벽 우회** | ✅ 가능 | ❌ 불가능 |
| **연결 속도** | 초기 연결 다소 느림 | 빠름 |
| **안정성** | 높음 (네트워크 문제 없음) | 네트워크에 따라 다름 |
| **사용 시나리오** | 방화벽/네트워크 문제 시 | 같은 네트워크에서 빠른 개발 |

---

## 터널 모드 사용 방법

### 기본 사용법

```powershell
cd app
npx expo start --tunnel
```

### 자동 터널 모드 설정 (권장)

`package.json`에 스크립트 추가:

```json
{
  "scripts": {
    "start": "expo start --tunnel",
    "start:lan": "expo start --lan",
    "start:web": "expo start --web"
  }
}
```

이렇게 하면 `npm start`만 실행해도 터널 모드로 시작됩니다.

## 터널 모드를 계속 사용해도 되나요?

### ✅ 사용해도 되는 경우

1. **네트워크 문제가 있는 환경**
   - 회사/학교 네트워크 (방화벽)
   - 공용 Wi-Fi
   - 다른 네트워크에 연결된 기기

2. **안정적인 연결이 필요한 경우**
   - 네트워크 문제로 인한 연결 끊김 방지
   - 일관된 개발 환경 유지

3. **개인 프로젝트 개발**
   - 성능 차이가 크지 않음
   - 편의성 우선

### ⚠️ 일반 모드가 더 나은 경우

1. **같은 네트워크에서 빠른 개발이 필요한 경우**
   - 팀 내부 개발
   - 로컬 네트워크에서 빠른 반응 필요

2. **대용량 파일 전송이 많은 경우**
   - 터널 모드는 Expo 서버를 거치므로 약간 느릴 수 있음

---

## 터널 모드의 장점

1. **네트워크 독립적**
   - PC와 모바일이 다른 네트워크여도 작동
   - 방화벽 문제 없음

2. **안정성**
   - 네트워크 변경 시에도 연결 유지
   - "Failed to download remote update" 오류 방지

3. **편의성**
   - 네트워크 설정 걱정 없이 개발 가능

---

## 터널 모드의 단점

1. **초기 연결 속도**
   - 첫 연결 시 약간 느릴 수 있음 (보통 10-30초)
   - 이후에는 일반 모드와 비슷

2. **Expo 서버 의존**
   - Expo 서버가 다운되면 연결 불가 (드묾)
   - 일반 모드는 로컬 네트워크만 사용

---

## 권장 사용 방법

### 개발 환경별 권장 설정

**개인 개발 (권장)**
```powershell
# 터널 모드를 기본으로 사용
npx expo start --tunnel
```

**팀 개발 (같은 네트워크)**
```powershell
# 일반 모드 사용 (더 빠름)
npx expo start --lan
```

**네트워크 문제 발생 시**
```powershell
# 언제든지 터널 모드로 전환 가능
npx expo start --tunnel
```

## package.json 설정 예시

터널 모드를 기본으로 사용하려면:

```json
{
  "scripts": {
    "start": "expo start --tunnel",
    "start:lan": "expo start --lan",
    "start:web": "expo start --web",
    "android": "expo start --android --tunnel",
    "ios": "expo start --ios --tunnel"
  }
}
```

이렇게 설정하면:
- `npm start` → 터널 모드로 시작
- `npm run start:lan` → 일반 모드로 시작 (필요 시)

---

## 주의사항

1. **Expo 계정 필요**
   - 터널 모드는 Expo 계정이 필요할 수 있습니다
   - 무료 계정으로도 사용 가능

2. **인터넷 연결 필요**
   - 터널 모드는 인터넷 연결이 필요합니다
   - 오프라인에서는 작동하지 않음

3. **개인 정보**
   - 터널 모드는 Expo 서버를 거치지만, 코드는 로컬에서 실행됩니다
   - 보안 문제 없음

---

## 결론

**터널 모드를 계속 사용해도 전혀 문제없습니다!**

- ✅ 안정적이고 편리함
- ✅ 네트워크 문제 없음
- ✅ 개발 생산성에 큰 영향 없음

특히 네트워크 문제로 고생했다면, 터널 모드를 기본으로 사용하는 것을 권장합니다.
